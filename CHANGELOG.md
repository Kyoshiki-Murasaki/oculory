# Changelog

## Unreleased — Phase 3: run isolation + safe model-validation workflow
Made larger model-driven runs safe from trace contamination, accidental approvals, and uncontrolled spend — without changing the scripted pipeline (`oculory experiment` still reproduces `meaningful_technical_success`, 72 traces, 100% pass rate). New run-management layer: `src/schema/run-manifest.ts`, `src/pipeline/run-context.ts` (run-id, path safety, `--clean`/`--append`/`--force` isolation semantics, manifest builder), `src/pipeline/run-store.ts` (`RunStore` = `Store` + manifest + `logs/` + `--force` reset). Each live run gets an isolated `.oculory/runs-live/<run-id>/` directory with a `manifest.json` (reproducibility metadata: run id, kind, model, provider, temperature, budget, trials, partition, git commit, node version, command). New first-class commands (`OPENAI_API_KEY` only; never auto-approve; fail closed on budget): `model-smoke`, `model-experiment`, `model-replay` (`src/pipeline/model-run.ts`, testable via injected `ModelClient`). `verify`/`mine`/`review`/`approve` accept `--run-dir` and stay inside it. Candidate `risk_profile` annotation (`src/pipeline/candidate-risk.ts`) drives review clarity and approval safety (`src/pipeline/approval.ts`): `approve --all-stable` blocks smoke-only / unstable / risky model candidates unless `--allow-smoke`/`--allow-unstable`/`--allow-risky` is passed, and records overrides (`approved_by`, `approval_mode`, `overridden_warnings`, `reason`). Expanded `doctor` (node_modules, stray `.env`, gitignore, git-tracked, key presence — never printed). `.gitignore` cleaned up. 43 new tests (89 total), all against a stubbed `ModelClient` or pre-network CLI validation — no test calls a real API; no live model run has been performed in this repository yet. New doc: `docs/24_RUN_ISOLATION_AND_MODEL_VALIDATION.md`.

## Unreleased — Phase 2: model-driven traffic (schema_version 2)
Added real model-backed traffic alongside the existing scripted pipeline, which is unchanged in behaviour. `AgentPolicy`/`recordSession`/`replaySuite`/`runExperiment` are now async. New `ModelPolicy` (`src/runner/model-policy.ts`) runs a controlled tool-call loop against an injectable `ModelClient`, with an `OpenAiClient` implementation (`OPENAI_API_KEY` only, read from the environment, never committed). `RawTrace.agent` gained `provider`/`model`/`tokens_in`/`tokens_out`/`cost_usd`/`budget_usd`; `RawTrace` gained `trial` (docs/04). Hard per-invocation budget guard, default $5, fails closed. Recording-time instability detection for `--trials N>1` (`src/pipeline/instability.ts`), explicitly distinguished from the still-unimplemented replay-time instability stub (docs/04). CLI: `--policy scripted|model|<id>`, `--model`, `--trials`, `--budget-usd`. 19 new tests (46 total), all against a stubbed `ModelClient` — no test calls a real API, and no real model run has been performed in this repository yet (docs/23). New doc: `docs/23_MODEL_TRAFFIC_VALIDATION.md`.

## 0.1.0 — 2026-07-04 (unreleased; private technical use)
First working MVP. Full pipeline: record → verify → normalize → mine → review → approve → suite → run → compare. SQLite demo MCP server (8 tools, stdio JSON-RPC, 10 injectable defects), deterministic outcome verifier, provenance-linked assertion miner with anti-overfitting rules (change-eligibility + corroboration), versioned suites, replay + deterministic evaluator, schema-smoke baseline proxy, Markdown/JSON regression reports, CLI with CI exit codes. 26 tests. Pre-registered internal experiment: mined precision 1.0 / recall 0.889, 4 unique behaviour-level detections beyond the proxy baseline, 100% unmutated pass rate. Known gates before public release: model-driven traffic, executed CI (docs/21, docs/22).
